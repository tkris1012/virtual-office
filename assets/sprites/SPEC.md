# スプライトシート規格（歩行アニメ・キャラクター用）

このフォルダにキャラクターのスプライトシートを追加する（AIが生成する場合も含む）際は、
**必ずこのファイルを最初に確認してから**作業してください。

Universal LPC Spritesheet Generator の標準書き出し形式に合わせています。
LPCで生成する場合も、オリジナルで描く場合も、同じ規格に従うことで
アプリ側のコード（`main.js` の `drawSpriteAvatar` 周り）を一切変更せずに
画像ファイルを追加・差し替えできます。

## 必須仕様

| 項目 | ルール |
|---|---|
| セルサイズ | 64×64px 固定 |
| 行の並び順（固定・変更不可） | 0行目＝上向き、1行目＝左向き、2行目＝下向き、3行目＝右向き |
| 列（コマ数） | 歩行サイクル9コマ（0〜8列目を使用）。9列より後ろに空白列があっても無視されるだけなので実害なし |
| 背景 | 必ずアルファチャンネル付きの本物の透過PNG（RGBA）。**スクリーンショット経由での保存は禁止**（透過が消える） |
| 足元の位置 | 各セルの中で、足元がセル下端付近の同じ高さに揃っていること（バラバラだと歩行時にガクガクする） |
| 画像全体のサイズ | 64pxのセルがきっちり並ぶサイズにする（576×256でも、LPCツールが吐き出す832×256でもどちらでもよい）。半端なサイズ・均等に割り切れないサイズは禁止 |
| 対応アニメーション | 現状は「歩行」のみ。攻撃/座り等の他アクションは不要 |

## 配置・命名ルール

```
assets/sprites/<character-id>/
  standard/walk.png      ← 本体（上記ルールに従ったスプライトシート）
  credits/                ← 描いた人の名前・ライセンスを記録（オリジナルでも推奨）
```

- `<character-id>` は英数字のスラッグにする（例：`male_01`、`female_knight`）。日本語ファイル名は
  git/ターミナル上での文字化け表示の原因になるため避ける。

## 作成前チェックリスト

- [ ] 保存前に画像編集ソフトで開き、背景が市松模様（透過）に見えるか確認した
  （スクリーンショットで保存すると透過が消えるので厳禁）
- [ ] 64×64pxのグリッドで、4行×9コマ（+空白列があってもよい）になっている
- [ ] 0行目=上, 1行目=左, 2行目=下, 3行目=右 の順序を守っている
- [ ] 各コマで足元の高さが揃っている
