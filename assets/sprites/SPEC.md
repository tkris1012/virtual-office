# スプライトシート規格（歩行アニメ・キャラクター用）

このフォルダにキャラクターのスプライトシートを追加する（AIが生成する場合も含む）際は、
**必ずこのファイルを最初に確認してから**作業してください。

Universal LPC Spritesheet Generator の書き出し形式に合わせています。
LPCで生成する場合も、オリジナルで描く場合も、同じ規格に従うことで
アプリ側の合成・描画コード（`main.js` の `drawSpriteAvatar` / `loadSpriteCharacter` 周り）を
一切変更せずに画像ファイルを追加・差し替えできます。ただし**新しいキャラクターを
選べるようにするには、画像を置くだけでなく `sprite-characters.js` への登録が必要**です
（下記「登録ルール」参照）。

## 必須仕様

| 項目 | ルール |
|---|---|
| セルサイズ | 64×64px 固定 |
| 行の並び順（固定・変更不可） | 0行目＝上向き、1行目＝左向き、2行目＝下向き、3行目＝右向き |
| 列（コマ数） | 歩行サイクル9コマ（0〜8列目を使用）。9列より後ろに空白列があっても無視されるだけなので実害なし |
| 背景 | 必ずアルファチャンネル付きの本物の透過PNG（RGBA）。**スクリーンショット経由での保存は禁止**（透過が消える） |
| 足元の位置 | 各セルの中で、足元がセル下端付近の同じ高さに揃っていること（バラバラだと歩行時にガクガクする） |
| 画像全体のサイズ | 64pxのセルがきっちり並ぶサイズにする（576×256でも、LPCツールが吐き出す832×256でもどちらでもよい）。半端なサイズ・均等に割り切れないサイズは禁止 |
| 対応アニメーション | 現状は「歩行」のみ。攻撃/座り等の他アクションは不要 |

## ファイル形式：単一シート or レイヤー分割（どちらでもよい）

キャラクター1体分は、次の**どちらかの形**で用意します。

### A. 単一シート（シンプル）

```
standard/walk.png   ← 1枚に完成した絵
```

### B. レイヤー分割（LPCの「パーツ別」書き出し。着せ替え可能な素材向き）

```
standard/walk/
  010 body_color__light_.png
  015 basic_shoes__black_.png
  020 formal_pants__charcoal_.png
  035 collared_formal_longsleeve__white_.png
  100 human_male__light_.png
  120 buzzcut__dark_brown_.png
```

- ファイル名の**先頭の数字が重ね順**（小さい順＝奥から手前へ描画。体→靴→ズボン→服→顔→髪、のように）
- 全レイヤーは**同じピクセルサイズ**（例：832×256）で揃える。サイズが違うレイヤーがあると重ね描画がズレる
- アプリ側は起動時に全レイヤーを読み込み、番号順に1枚のCanvasへ自動合成してから使う（`main.js` の `loadSpriteCharacter`）ため、追加のコード変更は不要

## 配置・命名ルール

```
assets/sprites/<character-id>/
  standard/walk.png または standard/walk/*.png   ← 本体（上記どちらかの形式）
  credits/                                        ← 描いた人の名前・ライセンスを記録（オリジナルでも推奨）
```

- フォルダ名は日本語でも動作します（実際に稼働中のキャラクターも日本語フォルダ名です）が、
  git/ターミナル上で文字化けして表示されることがあるため、**可能なら英数字スラッグ**
  （例：`male_01`、`female_knight`）を推奨します。

## 登録ルール（重要）

画像を `assets/sprites/` に置いただけでは選択肢に出てきません。
**`sprite-characters.js` の `SPRITE_CHARACTERS` 配列に1エントリ追加**してください。

```js
{
  id: "英数字の一意なID",
  label: "設定画面に出す表示名",
  folder: "実際のフォルダパス（assets/sprites/以下）",
  layers: ["ファイル名を重ね順（昇順）で列挙"], // 単一シートなら要素1つでよい
}
```

登録すると、プロフィール設定の「マイアバター」→ アバター編集画面の「プリセット」に自動で表示されます。

## パーツライブラリ（`assets/sprites/parts/` ・ `sprite-parts.js`）

`sprite-characters.js` は「完成済みの1体」を丸ごと登録する仕組みですが、それとは別に
**個別の装飾アイテム（髪型・トップス・ボトムス・靴・アクセサリーなど）を1点ずつ管理する
ためのライブラリ**を `assets/sprites/parts/` に用意しています。将来的にアイテムを1点ずつ
取得・着せ替えできるようにする機能はこちらを土台にする想定です（**現時点ではまだ
どの画面からも参照されておらず、素材置き場として先行して用意している段階**です）。

```
assets/sprites/parts/
  body/           体型・素体（肌色・顔など）。zPrefix の目安は 10・100 番台
  hair/           髪型。120番台
  tops/           トップス（シャツ・ブラウス・上着）。35・55番台
  bottoms/        ボトムス（ズボン・スカート）。20番台
  shoes/          靴。15番台
  accessories/    ネクタイ・武具など。65・90・140番台など
```

- 各ファイルは LPCレイヤーPNG **1枚**（`sprite-characters.js` の `layers` 配列の1要素と同じもの）
- ファイル名に重ね順の数字は付けない。重ね順は `sprite-parts.js` の各エントリが持つ
  `zPrefix` フィールドで管理する
- 登録は `sprite-parts.js` の `SPRITE_PARTS` 配列に1エントリ追加する

```js
{
  id: "英数字の一意なID",
  category: "body | hair | tops | bottoms | shoes | accessories",
  bodyType: "male | female | skeleton など（体型が合わないと位置がズレるため必須）",
  label: "画面に出す表示名",
  file: "parts/以下の相対パス（例: hair/buzzcut__dark_brown_.png）",
  zPrefix: 重ね順の数値（小さいほど奥）,
}
```

- **単一シート形式（`standard/walk.png` 1枚だけ）のキャラクターはパーツに分解できません**。
  パーツライブラリに載せたい場合はレイヤー分割形式で用意してください
  （例：`male_casual` は単一シートのため未登録）
- 同じ見た目のファイルが複数キャラクターに重複して存在する場合は、内容が同一かどうか
  （`md5sum` などで）確認してから1つに集約する。**体型が違うと同名ファイルでも中身が
  別物**なので注意（例: 男性用と女性用の `body_color__light_.png` は別物）

## 作成前チェックリスト

- [ ] 保存前に画像編集ソフトで開き、背景が市松模様（透過）に見えるか確認した
  （スクリーンショットで保存すると透過が消えるので厳禁）
- [ ] 64×64pxのグリッドで、4行×9コマ（+空白列があってもよい）になっている
- [ ] 0行目=上, 1行目=左, 2行目=下, 3行目=右 の順序を守っている
- [ ] 各コマで足元の高さが揃っている
- [ ] レイヤー分割の場合、全レイヤーのピクセルサイズが一致している
- [ ] `sprite-characters.js` に登録した
- [ ] （個別パーツとしても使う場合）`assets/sprites/parts/` にカテゴリ別で配置し、
      `sprite-parts.js` に登録した
