# 戦闘システム仕様案

## 1. 目的

スケルトンの攻撃、二段階HP、呪いによる衣装変化、PK連続記録と二つ名を、ルーム滞在中の軽量なゲーム要素として提供する。

本仕様ではサーバー側の命中計算を行わず、攻撃者側が命中対象を判定し、対象者側が自分のHPを更新する。既存の「各ユーザーは自分のpresenceだけを書き込める」というRealtime Databaseの権限構造を維持する。

## 2. 基本用語

| 用語 | 意味 |
|---|---|
| 通常状態 | 本来の衣装で行動している状態 |
| スケルトン状態 | スケルトン専用HPと攻撃能力を持つ状態 |
| 呪い | 通常HPが0になったことで強制的にスケルトン化した状態 |
| 選択スケルトン | ユーザー自身が衣装としてスケルトンを選択した状態 |
| PK | 相手の現在有効なHPゲージを0にすること |
| PK連続数 | 現在のスケルトン状態中に成功したPK数 |

## 3. HP

### 3.1 HP定義

| 状態 | 最大HP | HPが0になった場合 |
|---|---:|---|
| 通常状態 | 100 | 呪われたスケルトンへ移行し、スケルトンHPを200にする |
| スケルトン状態 | 200 | スケルトン状態を解除し、復帰先衣装と通常HP100へ戻す |

通常HPとスケルトンHPは別の値として管理する。一度に画面表示するHPバーは、現在の状態に対応する一本だけとする。

### 3.2 状態遷移

通常衣装から呪われた場合:

```text
通常衣装・通常HP 100
    │ 通常HPが0
    ▼
呪われたスケルトン・スケルトンHP 200
    │ スケルトンHPが0
    ▼
スケルトン化直前の衣装・通常HP 100
```

ユーザーが最初からスケルトンを選択していた場合:

```text
選択スケルトン・スケルトンHP 200
    │ スケルトンHPが0
    ▼
デフォルト衣装・通常HP 100
```

デフォルト衣装には `DEFAULT_SPRITE_CHARACTER_ID` を使用する。現行定義では「男性・ネイビースーツ」である。

### 3.3 退出・再入室

- HPと戦闘状態はルーム滞在中のみ有効とする。
- 退出時に通常HP、スケルトンHP、PK連続数、二つ名、呪い状態をリセットする。
- 呪いによるスケルトン衣装はプロフィールへ保存しない。
- 呪われた状態で退出した場合、再入室時はプロフィールに保存されている本来の衣装と通常HP100で開始する。
- プロフィール上でスケルトンを選択している場合、再入室時は選択スケルトン、スケルトンHP200、PK連続数0で開始する。

## 4. HPバー表示

### 4.1 配置

- 現在のHPバー位置から、HPバー本体の太さと枠線を合わせた分だけ上へ移動する。
- 現行の高さ4px、上下枠線各1pxを前提として、移動量は6pxとする。
- 上から「HPバー」「表示名」の順に配置し、互いに重ならないようにする。

### 4.2 色

通常HPバー:

- HP比率が50%より大きい: 緑
- HP比率が25%より大きく50%以下: 黄
- HP比率が25%以下: 赤

スケルトンHPバー:

- 紫系の専用色を使用する。
- 残量による色変化を設ける場合も、通常HPと区別できる紫系統を維持する。

## 5. スケルトン化と衣装復帰

### 5.1 スケルトン化の原因

スケルトン状態へ入る際、原因を次のいずれかで記録する。

| 値 | 意味 |
|---|---|
| `curse` | 通常HPが0になり、強制的にスケルトン化した |
| `selected` | ユーザー自身がスケルトン衣装を選択した |

### 5.2 復帰先

- `curse`: スケルトン化直前の衣装へ戻す。
- `selected`: `DEFAULT_SPRITE_CHARACTER_ID` の衣装へ戻す。
- 復帰時に通常HPを100へ戻す。
- スケルトンHP、PK連続数、二つ名をリセットする。
- 呪われた表示を解除する。

### 5.3 衣装変更

- 呪いによるスケルトン状態では衣装変更を禁止する。
- 呪いはスケルトンHPが0になるまで解除できない。
- ユーザー自身が選択したスケルトンは、衣装画面から任意に解除できる。
- 選択スケルトンを手動解除した場合も、PK連続数と二つ名をリセットする。

## 6. 表示名

### 6.1 呪われた表示

スケルトン状態では、マップ上の表示名を次の形式にする。

```text
呪われた {二つ名}{ユーザー名} 💀×{PK連続数}
```

例:

```text
呪われた 魂狩りの山田 💀×3
```

表示ルール:

- `呪われた` の文字だけ紫色にする。
- 二つ名とユーザー名は従来の名前色を使用する。
- プロフィールに保存された表示名自体は変更しない。
- PK連続数が0の場合、どくろアイコンと数値は表示しない。
- 呪いが解除された場合、通常の表示名へ戻す。
- 表示の統一性を優先し、呪いと選択のどちらであっても、スケルトン状態では `呪われた` を表示する。

## 7. PK連続数

### 7.1 加算条件

- 攻撃者がスケルトン状態であること。
- 対象者の現在有効なHPゲージを0にしたこと。
- 通常HPを0にして対象をスケルトン化した場合も1PKとする。
- スケルトンHPを0にして対象を通常衣装へ戻した場合も1PKとする。
- 一回の攻撃で複数人のHPを0にした場合、撃破人数分を加算する。

### 7.2 リセット条件

- スケルトンHPが0になり、通常衣装へ戻った。
- 選択スケルトンを手動で解除した。
- ルームから退出した。

PK連続数はプロフィールへ保存しない。別のスケルトン状態へ持ち越さない。

### 7.3 表示

- PK連続数が1以上の場合、名前の末尾に `💀×{件数}` を表示する。
- 単純な `x` だけの表示は使用しない。
- 例: `💀×1`、`💀×3`、`💀×100`

## 8. 二つ名

現在のスケルトン状態中のPK連続数に応じて二つ名を付与する。複数の条件を満たす場合、到達済みの最上位の二つ名だけを表示する。

| PK連続数 | 二つ名 | 表示例 |
|---:|---|---|
| 0～2 | なし | `呪われた 山田 💀×2` |
| 3～9 | 魂狩りの | `呪われた 魂狩りの山田 💀×3` |
| 10～49 | 冥界を歩む | `呪われた 冥界を歩む山田 💀×10` |
| 50～99 | 死者の軍勢を率いる | `呪われた 死者の軍勢を率いる山田 💀×50` |
| 100以上 | 百魂を従えし | `呪われた 百魂を従えし山田 💀×100` |

二つ名はPK連続数から表示時に算出し、独立した永続データとして保存しない。

## 9. 攻撃と命中判定

### 9.1 攻撃条件

- スケルトン状態のプレイヤーだけが攻撃できる。
- `L` キーで斧攻撃を開始する。
- 攻撃中の再入力とキー長押しによる連打を無効にする。
- 入力欄やモーダル操作中は発動しない。

### 9.2 軽量命中判定

- 攻撃者のクライアントが攻撃開始時の対象を判定する。
- 攻撃者の正面64px、横幅64pxを攻撃範囲とする。
- 攻撃者と異なるエリアにいるプレイヤーは対象外とする。
- 命中対象のユーザーIDを攻撃アクションに含めて共有する。
- 対象者は受信した対象IDに自分が含まれていた場合、自分のHPを減らす。

本方式は軽量なクライアント判定であり、不正操作への完全な耐性は持たない。将来、競技性や報酬を導入する場合はサーバー側判定への移行を検討する。

## 10. 同期データ

### 10.1 プレイヤー戦闘状態

想定するpresence内のデータ構造:

```text
rooms/{room}/players/{uid}/combat
├─ phase: "normal" | "skeleton"
├─ normalHp: 0..100
├─ skeletonHp: 0..200
├─ skeletonOrigin: null | "curse" | "selected"
├─ returnAppearance
│  ├─ spriteCharacterId
│  └─ spriteParts
├─ pkStreak: 0以上
└─ lastDefeat
   ├─ id
   ├─ attackerId
   ├─ attackId
   └─ at
```

`returnAppearance` は `curse` の場合にだけ使用する。プロフィールを直接変更せず、スケルトン解除時の復帰先として保持する。

### 10.2 攻撃アクション

```text
rooms/{room}/players/{uid}/runtime/action
├─ id
├─ type: "skeleton_slash"
├─ facing: "up" | "left" | "down" | "right"
├─ startedAt
└─ targets
   └─ {targetUid}: true
```

- 攻撃開始時に一回だけ送信する。
- アニメーションの各コマは送信しない。
- 受信側はサーバー時刻と `startedAt` の差から表示するコマを計算する。
- 同じ `id` のアクションは一度だけ処理する。

### 10.3 PK成功通知

対象者はHPが0になった際、自分の `lastDefeat` を更新する。

```text
lastDefeat:
  id: 一意な撃破ID
  attackerId: 攻撃者UID
  attackId: 原因になった攻撃ID
  at: サーバー時刻
```

攻撃者は他プレイヤーの `lastDefeat` を受信し、次の条件を満たした場合に自分のPK連続数を1増やす。

- `attackerId` が自分のUIDと一致する。
- 未処理の `id` である。
- 自分が現在もスケルトン状態である。

PK連続数は攻撃者本人のpresenceに書き込む。これにより、既存の「本人だけが自分のデータを書き込める」ルールを維持する。

## 11. 初期値

| 項目 | 初期値 |
|---|---:|
| 通常HP | 100 |
| スケルトンHP | 200 |
| 斧攻撃ダメージ | 25 |
| 攻撃正面距離 | 64px |
| 攻撃横幅 | 64px |
| PK連続数 | 0 |
| 通常衣装からのスケルトン原因 | `curse` |
| ユーザー選択スケルトン原因 | `selected` |

## 12. 手動確認項目

1. 通常衣装のHPが100で表示される。
2. HPバーが従来位置から6px上に表示される。
3. 通常HPが0になるとスケルトンHP200へ切り替わる。
4. スケルトン化直前の衣装が復帰先として保持される。
5. `呪われた` だけが紫色で表示される。
6. スケルトンの攻撃が他ユーザーへ同期される。
7. 正面の範囲内だけに攻撃が命中する。
8. HPを0にした攻撃者のPK連続数が増える。
9. PK連続数3、10、50、100で適切な二つ名へ切り替わる。
10. スケルトンHPが0になると本来の衣装へ戻る。
11. 最初からスケルトンだった場合はデフォルト衣装へ戻る。
12. スケルトン解除時にPK連続数と二つ名がリセットされる。
13. 退出・再入室で戦闘状態がリセットされる。
